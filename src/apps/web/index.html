图文语境违规检测 · 级联流水线

读出图文之间的潜台词

一张图可能正常，一句话可能只是擦边——但图 + 文 + 语境合在一起，可能正是违规。这是纯视觉模型、关键词库、甚至纯文本大模型共同的盲区。Subtextor 用一条级联流水线，把昂贵的多模态判断只用在刀刃上。

核心立意 · 同图异文

同一张二维码，只换一句配文，判定就该相反。

同一张图
配文 A扫码进群，兼职日结，日入三百+ → ✕拦截 · 诈骗导流
配文 B活动现场签到码，扫码领取纪念册 → ✓放行 · 正常

二维码本身是中性的——是配文的语境决定了它违规与否。纯关键词挡不住变体、纯视觉会一刀切误伤，只有图文联合的多模态判别才读得懂这层潜台词。这也是项目唯一会硬拦的"确定性证据"（已知恶意网址 / 露骨画面）之外、所有判定的核心立意。

3VLM只判拿不准的少数帖，图文联合给理由
4人工待人工裁决写回缓存，形成闭环

同一张图，配文不同，判定就该不同 —— 下面是它从输入到判定要走的每一级。

1

哈希匹配 Hash Match

命中就直接复用上次结论

给图算一个「图像指纹」、给配文算一个「文字指纹」，两者合成一把钥匙。以前判过的同一张图配同样的文，直接复用上次结论、不再往下跑；同一张图换了配文，就不算命中。

↓ 仅哈希未命中的帖，才进入初筛
2

初筛 Prefilter

逐帖都跑 · 要快
IN未命中的帖
→→
图像侧
NSFW 视觉 只看图，识别色情等露骨画面 · 常开
二维码解码 找出图里的二维码并解出内容（网址 / 名片 / 纯文字）
文本侧
OCR 图内文字 把图里印着的字识别出来，并入配文
→
文字粗筛 对「配文 + 图内文字」做关键词规则，并挑出其中的网址
→
URL 信誉 把（二维码 + 文字里的）网址跟已知恶意名单核对
→
判定门 · 按优先级
1露骨画面 / 命中恶意网址→ 直接拦截
2只发现线索（二维码 / 可疑话术 / 陌生网址）→ 进入第三级 VLM
3什么都没发现→ 放行
↓ 仅初筛拿不准（判定门第②种：只有线索）的少数帖，才召唤 VLM
3

VLM 图文联合判别 Joint

只判前面拿不准的少数帖

把图像、配文（含 OCR 抽出的字）和初筛收集到的线索一起交给 VLM（看得懂图的大模型），让它读懂图文背后的真实意图，给出违规程度：明确 / 较可能 / 模棱两可。

↓ VLM 给出标签 + 违规程度，进入分流
OUT
